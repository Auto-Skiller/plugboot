# 🧭 Navigator Protocol (Agentic Sync)

**Role:** Synchronize `.brain/.toolbox_library/` and `toolbox_library.router.yaml`.

## When to Execute
- At the start of a Session.
- Whenever a new Toolbox, Skill, or Agent is added or modified in the library.

## Execution Steps

### 1. Enforce Physical Schemas
Scan `.brain/.toolbox_library/` (both `core.toolbox` and `extended.toolbox`).
Every toolbox YAML file MUST strictly adhere to the schema below:

```yaml
# 🧰 TOOLBOX: [Name]

metadata:
  name: string
  description: string       # Copied from actual file/content. Never generic placeholders.
  when_to_use: string       # Generated by agent from description. Specific and actionable.

content:
  agents: []                # Array of objects: {name, path, description, when_to_use}
  skills:                   # Full frontmatter from SKILL.md must be copied here
    - name: string
      path: string
      description: string
      version: string
      triggers: [string]
      inputs: [string]
      outputs: [string]
      cataloger_lock: bool
      when_to_use: string
  relevant_toolboxes: [string]
```

*Fix any toolbox files that do not match this schema.*

### 2. Update the Router Map
Open `.brain/meta.router/toolbox_library.router.yaml`.
Ensure the `core_toolboxes:` and `extended_toolboxes:` blocks perfectly map all physical paths.
It MUST adhere to the `expected_schema` defined in `meta.router.yaml`, including:
- `path`, `yaml_path`, `description`, `when_to_use`
- `agent_count`, `agent_names` (list of .md filenames)
- `skill_count`, `skill_names` (list of skill folder names)

Recalculate counts and names whenever agents or skills are added or removed.
