# 🏗️ Projects Sync Protocol (Agentic OS v5)
> **Schema Version:** 1.0 | Canonical source of truth: `projects/` → `projects.yaml`

**Role:** Automatically discover and catalog standalone codebases in the `projects/` directory.

## 📦 File Schema (Enforced)

### Project Index Schema (`projects.yaml`)
The structure generated by the engine for each project:
```yaml
[project_id]:
  path: "projects/[name]/"
  readme: "projects/[name]/README.md"
  entry_point: string          # e.g., index.js, main.py
  type: script | web_app | api | other
  stack: [string]
  status: active | paused | completed | archived
  description: string          # Auto-extracted from first paragraph of README
  dependencies:
    runtime: string            # path to requirements.txt or package.json
```

---

## ⚙️ Execution Steps (Protocol)

### 1. Discovery
- Scan the `projects/` root for sub-directories.
- Ignore folders starting with `.` or `_`.

### 2. Metadata Extraction
- Read `README.md` to extract the project description.
- Detect `requirements.txt` (Python) or `package.json` (JS) to determine the tech stack.
- Identify potential entry points.

### 3. Cataloging
- Update `projects.yaml` with the discovered projects and their metadata.
