# security_and_boundaries

**Pillar:** agent_operating_conventions  **Aspect:** Architecture

## Role
Hard limits on what the agent may and may not do for this user (out-of-scope + security posture).

## Functionalities
- Reject out-of-scope requests per OUT-OF-SCOPE
- Apply the security rules in SECURITY.md
- Stop before any boundary-violating action

## when_to_use
When a task maps to the function(s) above.

## triggers
- when a task requires security and boundaries
- when activating a capability housed in this group
