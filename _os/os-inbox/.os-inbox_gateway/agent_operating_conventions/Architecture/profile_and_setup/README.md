# profile_and_setup

**Pillar:** agent_operating_conventions  **Aspect:** Architecture

## Role
The user's identity, environment, and one-time setup the agent must know before acting.

## Functionalities
- Onboard a new session: read mandatory-initial-read
- Know the user profile + machine/workspace setup
- Honor setup constraints when configuring tooling

## when_to_use
When a task maps to the function(s) above.

## triggers
- when a task requires profile and setup
- when activating a capability housed in this group
