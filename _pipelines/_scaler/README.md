# ⚖️ Scaler Pipeline: System Assimilation & Evolution

## Objective
The Scaler is the primary metabolic engine of the **Agentic OS v5.1**. Its purpose is to ingest external discoveries (tools, skills, logic, business strategies) and internal architectural needs, then strategically architect and integrate them into the workspace while preserving the foundational OS DNA.

---

## 🏛️ Pillar-Based Architecture
The pipeline is organized into three distinct pillars of evolution:

1.  **`Foundational_Integrity`** (Stability): Core OS systems, identity laws, routing schemes, and structural blueprints.
2.  **`Operational_Muscles`** (Power): The `.toolbox_library`, automation scripts, agentic skills, and capability engineering.
3.  **`Value_Generation`** (Growth): Monetization logic, business strategies, and value-driven market deployments.

---

## 🧠 Strategic Intelligence (v5.1)
The Scaler now operates using a **Profile-Based Control** system defined in `CONTROLER.yaml`:

*   **Mode-Specific Profiles**: Distinct behaviors for `INTERNAL` (high-autonomy architectural growth) vs `EXTERNAL` (strict review-gated discovery).
*   **Action Gates**: Every integration action is classified into either **`EXECUTION`** (Auto-Integrated) or **`PLANNING`** (Awaiting Manual Approval).
*   **Integration Types**:
    *   **INJECT**: Append missing logic to existing files.
    *   **UPGRADE**: Modernize existing components while preserving foundational DNA.
    *   **BUILD_NEW**: Create and register entirely new components.
    *   **EXTEND**: Add sibling features via cohesive expansion patterns.

---

## 🚦 Operational Flow
1.  **Discovery**: Items land in the `_EXTERNAL_INBOX` or are identified via internal architecture audits.
2.  **Classification**: Items are routed to the correct Pillar via **Utility-First Routing**.
3.  **Mapping & Analysis**: Perform Double-Scan and Strategic Interrogation to identify target files and dependencies.
4.  **Proposing**: Generate **Proposal/Action Cards** in the relevant Pillar folder.
5.  **Gatekeeper Check**: The agent checks the `CONTROLER.yaml` profile. If the action is in `EXECUTION`, it proceeds. If in `PLANNING`, it posts to the review queue.
6.  **Integration**: Execute changes, perform Verification Scans, and trigger the master `.meta_sync` engine.

---

## 📂 Structure Overview
*   **`Foundational_Integrity/`**, **`Operational_Muscles/`**, **`Value_Generation/`**: Active Pillar workspaces.
*   **`.scaler_brain/`**: Isolated operational logic (Runbooks, Trackers, Scratchpad).
*   **`.scaler_runtime/`**: Ephemeral execution data for the pipeline.
*   **`_EXTERNAL_INBOX/`**: Central landing zone for all incoming data.
