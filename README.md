# Autonomous Workspace

**Generate Revenue with Minimal Human Intervention.**

Autonomous Goals-powered workspace for AI agents (Claude Code, Cursor, Codex, Jules, Antigravity, OpenClaw) to Amplify human vision so they can focus on strategy and direction.

Agents can manage projects and build products using a pipeline system and massive powers (agents, RAG knowledge, skills, Executable commands, Coding rules, Reusable templates, Automation scripts ...)

## Core Philosophy

This workspace is designed for **autonomous operation**. Human operators set strategic direction and goals, while AI agents execute autonomously to deliver revenue-generating products and services.

### Key Principles

- **Amplification, Not Replacement** - Humans focus on vision, strategy, and direction. Agents handle execution, research, and production.
- **Revenue-Focused** - Every pipeline, project, and capability ties back to generating income with minimal ongoing human intervention.
- **Agent-Native** - Built for AI agents to operate independently with clear behavioral protocols, decision-making frameworks, and operational modes.

---

## Quick Start

### For AI Agents

1. **Read `AGENTS.md`** - Understand your persona, operational modes, and behavioral protocols
2. **Read `INDEX.md`** - Navigate the workspace structure and find relevant resources
3. **Check `BOARD.md`** - Detect current mode and active goals

### For Human Operators

1. Set strategic goals in `BOARD.md`
2. Switch to appropriate mode (NORMAL/COLLAB/AUTO)
3. Let agents execute autonomously

## Workspace Structure

```
open-workspace/
|
├── AGENTS.md           # Agent identity, behaviors, and operational protocols
├── INDEX.md            # Navigation index and structure reference
├── BOARD.md            # Real-time session board for goals and mode tracking
│
├── _projects/          # Projects OS - project management
├── _pipelines/         # Pipeline OS - revenue product management
│
├── agents/             # AI role definitions (domain-partitioned)
├── knowledge/          # RAG knowledge base (architecture, domain guides)
├── skills/             # Domain capabilities and workflows
├── commands/           # Executable prompts (slash commands)
├── rules/              # Coding standards (language-prefixed)
├── templates/          # Reusable scaffolding templates
├── scripts/            # Automation and workspace utilities
│
└── _archive/           # Archived content (preserved history)
```

## Agent Operational Modes

| Mode | Indicator | Description |
|------|-----------|-------------|
| **NORMAL** | User is directing | Execute explicit commands, ask for clarification |
| **COLLAB** | User is partnering | Collaborate on decisions, propose and review |
| **AUTO** | User is absent | Act autonomously toward preset goals |

## Capabilities

Agents in this workspace can:

- **Plan** - Analyze requirements, draft strategies, evaluate trade-offs
- **Execute** - Implement features, fix bugs, ship products
- **Research** - Web search, documentation analysis, trend synthesis
- **Code** - Generate, review, and refactor across all languages
- **Automate** - Orchestrate multi-step workflows and pipelines
- **Document** - Capture decisions, write ADRs, maintain knowledge

## Getting Started with Pipelines

Pipelines are revenue-generating product workflows that run autonomously:

1. Define pipeline goal in `_pipelines/PIPELINES.md`
2. Agent reads pipeline spec and executes phases
3. Pipeline produces output (content, products, services)
4. Revenue flows with minimal human touch
