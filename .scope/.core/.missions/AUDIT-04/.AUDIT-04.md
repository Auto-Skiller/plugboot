# Toolbox Architecture Audit

## 1. Directory & File Overview
- The `.toolbox/` directory contains four main capability domains: `business_toolbox`, `engineering_toolbox`, `life_toolbox`, and `studio_toolbox`.
- Each toolbox contains multiple sub-domains (e.g., `business_toolbox/marketing`, `engineering_toolbox/frontend`).
- Every sub-domain contains an `agents/` and a `skills/` directory.
- `tree .toolbox` reveals there are 164 directories but exactly **0 files**. The toolboxes are structurally sound but entirely empty of content.
- `toolbox.rules.yaml` defines the standard for these files:
  - Agents must be `.md` files (`kebab-case.md`) with specific frontmatter.
  - Skills must be nested inside a namespace-prefixed directory (e.g., `[toolbox].[domain].[skill-name]/`) containing at least a `SKILL.md` file with specific frontmatter including dependencies and I/O contracts.

## 2. Gap Analysis
- **Missing Content:** The `.toolbox/` directory is entirely devoid of actual skills and agents. While the scaffolding is perfect, the muscles are missing.
- **Rule Mismatch from Knowledge Discovery:** As noted in `.scope/.core/.knowledge/knowledge.md` during AUDIT-02, the `.brain/.toolbox.context_control/toolbox.rules.yaml` dictates skills should be `.md` files. However, the current rules file says `skills` should be a **directory** structure containing a `SKILL.md` file. The documentation in the core knowledge base contradicts the current `toolbox.rules.yaml` schema.
- **Namespacing Complexity:** The rule for skill directories requires a strict `[toolbox].[domain].[skill-name]` prefix, which makes directory names extremely long and verbose.

## 3. Permanent Solutions
- **Resolve Knowledge Contradiction:** Update `.scope/.core/.knowledge/knowledge.md` to reflect the updated `toolbox.rules.yaml` (that skills are directories with `SKILL.md`, not just standalone `.md` files).
- **Update Core Engines for New Rules:** Ensure the `cataloger.engine.md` (which currently expects standalone `.md` files according to `knowledge.md`) is updated to scan for `SKILL.md` inside namespaced directories within the `skills/` folders.
- **Populate Core Skills:** Begin creating essential `SKILL.md` definitions and agent files so the Router Engine has targets to route to.
