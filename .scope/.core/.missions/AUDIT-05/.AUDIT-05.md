# Hustler Pipeline Audit

## 1. Directory & File Overview
- **Workspace:** `_pipelines/hustler/` contains a bunch of raw files in `_discoveries/_not_routed/` and a project folder `algerian-ecommerce/` with its discovery folder and a tracking file `algerian-ecommerce-PRODUCTS.yaml`.
- **Scope Context:** `.scope/pipelines/hustler/` contains `.knowledge/` with architecture, discovery, ingestion, processing, and pointer files.
- The `architecture.md` file defines a tracking system relying on `.yaml` / `.yalm` files (`PIPELINE-FOCUSES.yalm`, `[focus-name]-PRODUCTS.yaml`, etc.) to track focuses, products, and features.
- The `hustler-pointers.md` file references a `_pipelines/hustler/HUSTLER-BOARD.yaml`, which does NOT exist based on our `tree` scan.
- The `architecture.md` also references an outdated global mission board path: `.missions/pipelines-missions/hustler_missions/Hustler_missions.yaml`, which violates the current OS design (`.scope/pipelines/hustler/.missions/` should be used instead).

## 2. Gap Analysis
- **Missing Local Board:** The `HUSTLER-BOARD.yaml` mentioned in pointers is missing. Either it needs to be created or the pointer needs to be updated to rely entirely on the root `BOARD.yaml` (which is standard for OS v5).
- **Outdated Mission Paths:** `architecture.md` points to an old `.missions` architecture instead of the new `.scope/pipelines/hustler/.missions/` design.
- **Inconsistent Extensions:** `architecture.md` mixes `.yaml` and `.yalm` extensions randomly when describing the tracking system.
- **Raw File Clutter:** `_discoveries/_not_routed/` has 24 raw text/PDF/md files that need to be ingested/processed by the pipeline but currently just sit there.

## 3. Permanent Solutions
- **Centralize Command:** Update `hustler-pointers.md` to remove references to `HUSTLER-BOARD.yaml` and enforce that all operational state goes through the root `BOARD.yaml`.
- **Fix Architecture Docs:** Update `architecture.md` to correct the missions path to `.scope/pipelines/hustler/.missions/` and fix the `.yalm` typos to standard `.yaml`.
- **Create Ingestion Mission:** Define a formal mission in `.scope/pipelines/hustler/.missions/definitions/` (e.g., `ingest-raw-discoveries.mission.yaml`) to route and process the files currently stuck in `_not_routed/`.
