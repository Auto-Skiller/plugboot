# Scaler Pipeline Audit

## 1. Directory & File Overview
- **Workspace:** `_pipelines/scaler/` contains `_discoveries/` and `_proposals/` directories.
- `_discoveries/` contains `_multi-level`, `architecture-level`, `business-level`, and `capabilitys-level`.
- Inside `_discoveries/_multi-level/`, there is an extensive nested directory structure containing over 500 directories and 1000+ files, mostly looking like old toolbox skills or raw codebase artifacts.
- `_proposals/` contains subdirectories for proposals related to various parts of the system (`AUTO_HUSTLER-proposals`, `AUTO_SKILLER-proposals`, `_brain-proposals`, etc.).
- **Scope Context:** `.scope/pipelines/scaler/` contains `.knowledge/` with `.scaler.md`, `discovery_to_proposal.md`, and `mission.md`.

## 2. Gap Analysis
- **Outdated Naming Conventions:** The documents in `.knowledge/` (like `mission.md` and `discovery_to_proposal.md`) frequently reference outdated structural elements from OS v4 or earlier (e.g., `open-workspace`, `_core`, `_departments`, `_tools`, `INDEX.json`, `discoveries.json`, `proposals.json`, `AUTO_SKILLER`, `AUTO_HUSTLER`).
- **Pipeline Name Mismatch:** The documents refer to the pipeline as "AUTO_SKILLER", but the OS v5 structure calls it "scaler".
- **Massive Artifact Dump:** The `_discoveries/_multi-level/` folder contains a massive dump of what looks like an old version of the `_toolbox` (complete with python scripts, slides, design rules, etc.). This pollutes the scaler pipeline.
- **Missing Registries:** The `ledger.json` referenced in the playbook for tracking processed discoveries doesn't match the OS v5 BOARD state tracking principles.

## 3. Permanent Solutions
- **Rewrite Knowledge Docs:** Update `mission.md`, `discovery_to_proposal.md`, and `.scaler.md` to reflect OS v5 architecture:
  - Rename `AUTO_SKILLER` to `scaler`.
  - Rename `AUTO_HUSTLER` to `hustler`.
  - Replace `_departments` and `_tools` references with `.toolbox`.
  - Replace JSON indexes with `.catalog.yaml` logic.
- **Relocate Artifacts:** The massive dump inside `_pipelines/scaler/_discoveries/_multi-level/` should be migrated into the actual `.toolbox/` where it belongs, as it currently contains actual code, SKILL.md files, and resources that perfectly match the `toolbox.rules.yaml` schema (which explains why the actual `.toolbox/` was completely empty in AUDIT-04).
- **Update Tracking:** Integrate the scaler's proposal tracking directly into the `.scope/pipelines/scaler/.missions/runs/` architecture, abandoning legacy JSON ledgers.
