# AUDIT-03: Missions System Architecture Audit 🎯

## Findings

1. **`.scope/.core/.missions/` Gaps:**
   - Missing `definitions/` directory.
   - Missing `runs/` directory.
   - Current `AUDIT-*` missions are placed directly at the root of `.scope/.core/.missions/` instead of being organized within `runs/` or `definitions/`.

2. **`.brain/.missions.context_control/` Gaps:**
   - Missing `definitions.missions.catalog.yaml` as defined in `Core_Architecture.md` and `.brain/.catalogs.index.yaml`.
   - Missing `runs/` directory.
   - `core.missions.catalog.yaml` is currently at the root of `.brain/.missions.context_control/` instead of being inside `runs/`.

## Permanent Solutions

1. Create the missing `definitions/` and `runs/` directories in `.scope/.core/.missions/`.
2. Move the existing `AUDIT-*` mission directories into `.scope/.core/.missions/runs/`.
3. Update `BOARD.yaml` to point the target paths of the `AUDIT-*` goals to the new locations within `runs/`.
4. Create `.brain/.missions.context_control/definitions.missions.catalog.yaml` with the appropriate `source` pointing to `.scope/.core/.missions/definitions/`.
5. Create `.brain/.missions.context_control/runs/` directory and move `core.missions.catalog.yaml` into it. Update its `source` and `rules_used` paths accordingly.
6. Find and update any references to `core.missions.catalog.yaml` across the system to reflect its new location.
