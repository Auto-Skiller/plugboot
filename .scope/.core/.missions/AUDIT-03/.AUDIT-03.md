# Missions System Architecture Audit

## 1. Directory & File Overview
- `.brain/.missions.context_control/` contains `missions.rules.yaml` which clearly defines the directory structure (`definitions/` and `runs/`), naming conventions (`kebab-case.mission.yaml`), and the lifecycle logic for missions.
- It also contains `core.missions.catalog.yaml`, and catalogs for `pipelines/hustler` and `pipelines/scaler`. All are currently empty (`total_entries: 0`).
- From `AUDIT-02`, we saw that `pipelines/hustler/.missions` and `pipelines/scaler/.missions` both contain `definitions/` and `runs/` directories, matching the `missions.rules.yaml` structure.
- However, `.scope/.core/.missions/` does not have `definitions/` and `runs/` subdirectories. It currently contains the AUDIT mission folders (`AUDIT-01` through `AUDIT-06`).

## 2. Gap Analysis
- **Core Missions Structure Discrepancy:** The `missions.rules.yaml` dictates that `.scope/**/.missions/` should contain `definitions/` and `runs/`. However, `.scope/.core/.missions/` does not follow this pattern; instead, it houses goal-specific directories directly (e.g., `AUDIT-01/`).
- **Missions Definition Standard:** The current `AUDIT` tracking files are `.AUDIT-XX.track.yaml` (or just missing formal mission definitions) instead of the required `kebab-case.mission.yaml`.
- **Incomplete Catalogs:** The catalogs in `.brain/.missions.context_control/` index the `.missions/` directories, but with the current non-standard structure in `.core`, the cataloger won't properly recognize the mission files.

## 3. Permanent Solutions
- **Standardize .core Missions:** Create `definitions/` and `runs/` inside `.scope/.core/.missions/` and move the `AUDIT-XX` specific artifacts into the `runs/` subdirectory as per the `Board_Guide.md` instructions (`.scope/.core/.missions/runs/[GOAL-ID]/round-[NNN]/`).
- **Convert Tracking to Mission YAML:** Create proper `.mission.yaml` definition files inside `.scope/.core/.missions/definitions/` for any system-level operational missions, ensuring they include required phases and routing hints.
- **Update Master Router Index:** Ensure `core.missions.catalog.yaml` logic is updated to parse only `.mission.yaml` files inside `definitions/` directories, ignoring temporary `runs/` artifacts.
